# 후투티 — 상담사를 위한 증상기반 가진단 도구

임상가·상담자를 위한 증상 소견 기반 가진단(참고용 후보) 보조 웹앱입니다.
DSM-5-TR의 18개 진단 범주에 걸쳐 81개 진단의 체크리스트를 제공하며,
내담자 소견을 입력하면 관련 있는 진단 후보와 근거를 정리해 보여줍니다.

빌드 과정이나 서버가 필요 없는 순수 HTML/CSS/JS 정적 페이지이며,
입력한 정보는 브라우저 메모리에만 유지되고 어디로도 전송되지 않습니다.

## 중요 — 반드시 읽어주세요

- 이 도구는 **DSM-5-TR(American Psychiatric Association, 2022)의 공식 콘텐츠가 아닙니다.**
  체크리스트 문항은 DSM-5-TR이 다루는 진단 범주·증상군·기간 기준 등의 개념을 참고하여
  **AI(Claude)의 도움을 받아 요약·재서술**한 것으로, 원문을 그대로 발췌하지 않았습니다.
- 위 재서술 내용은 **자격을 갖춘 임상 전문가의 검증을 거치지 않았습니다.** 진단명, 항목 수
  기준(예: "9개 중 5개 이상"), 기간 조건 같은 세부사항에 오류가 있을 수 있으므로, **정확한
  원문 대조는 반드시 DSM-5-TR 자체를 참고**해야 합니다.
- 결과는 어디까지나 **선별(screening)을 위한 참고 정보**이며 진단이 아닙니다. 실제 진단은
  자격을 갖춘 임상가가 면담, 병력 청취, 감별진단 등을 포함한 종합적 임상 평가를 거쳐
  내려야 합니다.
- 자살사고, 자해, 급성 정신병적 증상 등 위기 징후가 있다면 체크리스트 결과와 무관하게
  즉시 전문적 위기평가가 필요합니다.
- 동료나 기관 내에 배포하실 경우, 저작권 및 임상적 책임 관련해 소속 기관의 법무·윤리
  절차를 확인하시길 권합니다. (자세한 배경은 아래 "저작권에 관하여" 참고)

## 사용 방법

### 로컬에서 바로 열기
`index.html` 파일을 브라우저로 열면 바로 사용할 수 있습니다.

### GitHub Pages로 배포
1. 이 폴더를 GitHub 저장소에 푸시합니다.
2. 저장소 **Settings → Pages**에서 배포 브랜치를 지정합니다 (예: `main`, 루트 디렉터리).
3. 몇 분 후 `https://<사용자명>.github.io/<저장소명>/` 주소로 접속할 수 있습니다.

### 로컬 서버로 실행 (선택)
```bash
python3 -m http.server 8000
# 이후 브라우저에서 http://localhost:8000 접속
```

## 사용된 색상

주 색상은 가톨릭대학교 상징색인 "가대 Blue"(Pantone 661 C, HEX #003594)를 기반으로
구성했으며, 보조색으로 화이트·그레이·실버 계열을 사용했습니다.

## 채점 로직 개요

각 진단은 DSM 방식의 "항목군(Criterion group)" 구조로 표현됩니다.

- 각 항목군에는 체크리스트 항목들과 그중 최소 몇 개를 충족해야 하는지(`min`)가 정의되어
  있습니다. (예: 주요우울장애는 9개 항목 중 5개 이상)
- `groupLogic`이 `AND`이면 모든 항목군을 동시에 충족해야 하고, `OR`이면 그중 한 항목군만
  충족해도 됩니다. (예: ADHD는 부주의형 또는 과잉행동-충동형 중 하나만 충족해도 해당)
- 기간 조건(`duration`)과 배제·부가 기준(`other`)은 체크박스로 표현하기 어려운 임상적
  판단이 필요한 부분이라 자동 채점에는 반영하지 않고 서술로만 안내합니다. 실제 판단 시
  반드시 함께 고려해야 합니다.
- 메인 페이지(`index.html`)의 "가진단 받기"는 키워드 겹침 비교 기반의 임시 결과입니다.
  "AI 분석 (Kanana)" 버튼을 누르면 브라우저 안에서 직접 실행되는 로컬 AI(Kanana GGUF,
  wllama)가 소견을 분석합니다 — 최초 1회 약 1.4GB 모델을 내려받으며, 이후에는 브라우저에
  캐시되어 빠르게 로드됩니다.

새 진단을 추가하거나 항목을 수정하려면 `js/data.js`의 배열에 같은 형식으로 객체를
추가·편집하면 됩니다.

## 저작권에 관하여

DSM-5-TR은 미국정신의학회(APA)의 저작물입니다. 진단명이나 "몇 개 중 몇 개 충족" 같은
사실적 정보는 저작권 보호 대상이 아니지만, 진단기준의 구체적 문장 표현은 APA의 저작물로
보호받습니다. 이 저장소의 체크리스트 문구는 원문을 그대로 복제하지 않고 AI의 도움을 받아
재서술했지만, 임상 전문가의 검증을 거치지 않았고 완전한 무위험을 보장하지 않습니다.
대규모 조직 배포나 상업적 활용을 고려하신다면 APA의 디지털 콘텐츠 라이선스 정책을
확인하시거나 법률 자문을 받아보시길 권장합니다. (본 안내는 법률 자문을 대체하지 않습니다.)
